What We Built: The PCBA BOM Agent

Our project aimed to solve a common problem in electronics manufacturing â€” how to analyze and optimize a BOM file quickly and intelligently. BOM files often contain dozens or hundreds of components, and finding cost-effective alternatives can be time-consuming and error-prone. We built a multi-agent system that does the heavy lifting automatically. Here's how it works:

  1. Input Processing Agent: It reads the BOM CSV file, extracts relevant fields (like MPN, description, manufacturer, quantity), and identifies which columns are usable or malformed.
  2. Part Analysis Agent: For each component, it queries the Octopart API to fetch real-time pricing, availability, and specs. It also looks for drop-in replacements â€” parts with matching specs that might offer better pricing or lead times.
  3. Optimisation Agent: This agent evaluates potential substitutes, considering criteria like cost savings, number of suppliers, and stock levels. It ranks these alternatives and flags components that could benefit from a swap.
  4. Report Generation Agent: Finally, a dedicated agent compiles all findings into a clean, human-readable report. This includes side-by-side comparisons, summaries of cost savings, and flagged issues.

Code for full agent workflow available open access on Github

Why Agentic AI?

Agentic AI goes beyond traditional automation. It's about giving systems autonomy to reason, plan, and act within a bounded goal. The system has both a brain and tools to use for action. This approach made perfect sense for our use case, where we needed to:

  • Chain multiple reasoning steps across structured data
  • Adapt to incomplete or ambiguous inputs
  • Respond to user prompts dynamically with new results
